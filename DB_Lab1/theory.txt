Что значат параметры подключения?
con. name - незвание - должно быть понятным, че оно значит, вроде запрещены 
зарезервированные слова. и еще в каких то версиях можно выделить его цветом
username - один из юзеров через которого идет подключение, обычно system\sys
если нужны права администратора. 
password - пароль от учетки, можно кликнуть его запомнить

con. type - тип, обычно выбирается бейсик, tns импортируется из tnsnames.ora, 
ldap связана со службами оракла, advanced - настройка собственного jdbc url 
дл коннекта с пользовательской бд

hostname - имя сервера на котором будет расположена бд, если локальный, то
localhost, иначе вводим url сервака
port - по дефолту оракл занимает 1251 порт
sid\service name: 
в версиях 12с и ниже выбирается SID - это тот системный айдишник, который мы 
указывали при установке оракла - должен быть orcl или чет подобное
в версиях выше используется service name, который можно найти в зав-сти от версии

на test протестится коннекшен и слева снизу выведется status: successful
на color можно настроить цвет как будет выглядить коннекшен в списке слева
